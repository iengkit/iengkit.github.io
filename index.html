A.I. Logics - Greedy WalkSAT (R)

This WALKSAT algorithm takes C and N as input It then returns a satisfying model or failure If a satifying model is returned then the program halts and output the number of flips required to return a satisfying model. A set of clauses are generated randomly per run for a specified number of runs dependent on N and C. Each numerical digit represents a symbol in clauses (for example, N is fixed at 20 hence {-20 .... -1} and {1.... 20} are used. The former set represents the negation of the N variables) with random assignment of true/false values represented as 0/1. If the initial assignment for a model is not satisfying then within a specified number of maxflips the program will try to find a model that satisfies the clauses by randomly selecting a clause that is false in the model, randomly chooses a symbol from that false clause then flip the value of that symbol in the model. The program then returns a satisfying model and output the number of flips required to generate the model if there is one. If no satisfying model within maxflips then no model will be returned, the program will terminate and output NA as the number of flips.
